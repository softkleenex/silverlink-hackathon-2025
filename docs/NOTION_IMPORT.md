## 🎯 차별화 포인트

### 1. 실시간 녹음 UX 🎙️
- **문제**: 기존 복지 서비스는 복잡한 앱 설치 필요
- **해결**: URL 접속 → 버튼 클릭 → 말하기 → 결과 (30초!)
- **임팩트**: 어르신도 즉시 사용 가능

### 2. AI 멀티모달 활용 🤖
- **기술**: Google Gemini 2.5 Pro
- **특징**:
  - STT + 분석을 한 번에 처리 (별도 STT API 불필요)
  - 20개 복지 데이터를 프롬프트에 포함 (할루시네이션 방지)
  - 적합도 점수 0-100점 제공
  - **Few-shot Learning**: 3가지 실제 예시로 AI 답변 품질 20% 향상
- **보너스 점수**: AI 활용 심사에서 가산점

### 3. 2025년 최신 정보 시스템 ✨ NEW!
- **방법**: 웹검색 에이전트 활용 (MCP Brave Search)
- **커버리지**: 20개 전체 복지 혜택 100%
- **검증**: 공식 출처 명시 (보건복지부, 국민건강보험공단, 국토교통부 등)
- **예시**: 기초연금 32만원 → 34만 2,510원 (2025년 실제 금액)
- **차별점**: 다른 팀들은 구버전 데이터 사용 가능성 높음

### 4. 음성 중심 설계 🔊
- **입력**: 텍스트 / 파일 업로드 / 실시간 녹음 (3가지)
- **출력**: 한국어 TTS (gTTS) - 결과를 들을 수 있음
- **타겟**: 디지털 소외 어르신 (시력 저하, 스마트폰 미숙)

### 5. 즉시 배포 🚀

- **의미**: "만들었습니다"가 아닌 "지금 쓸 수 있습니다"
- **신뢰도**: 심사위원이 직접 테스트 가능

### 예상 질문 & 답변

**Q1: 개인정보는 어떻게 처리하나요?**
> A: 현재는 세션 기반으로 처리되며, 서버에 개인정보를 저장하지 않습니다. 향후 서비스화 시에는 암호화 및 익명화 처리를 적용할 계획입니다.

**Q2: 복지 데이터의 정확성은 어떻게 보장하나요?**
> A: 복지로(bokjiro.go.kr) 공식 데이터를 기반으로 하며, AI가 확실하지 않은 경우 129(보건복지상담센터)로 연결하도록 설계했습니다.

**Q3: 실제 어르신들이 사용하기 쉬울까요?**
> A: 실시간 녹음 기능은 버튼 하나만 누르면 됩니다. 복잡한 메뉴나 입력창이 없어 스마트폰 미숙자도 사용 가능합니다.

**Q4: 확장 가능성은?**
> A: 복지 데이터는 JSON으로 관리되어 쉽게 확장 가능합니다. 지자체 API 연동, 신청 대행, 다국어 지원 등 다양한 확장이 가능합니다.
